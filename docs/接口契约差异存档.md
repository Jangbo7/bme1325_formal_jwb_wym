# 接口契约差异存档

**日期**：2026-05-15

## 目的

记录当前实现中，已经识别出的《全院统一接口契约 v1.0》差异项，作为后续整改依据。

本文档只做存档，不代表当前实现已经完全合规。

## 范围

- `backend/app/config.py`
- `backend/app/agents/patient_agent/*`
- `backend/app/agents/triage/*`
- `backend/app/agents/internal_medicine/*`
- `backend/app/agents/icu_doctor/*`
- `backend/app/api/routes/patient_agent_debug.py`

## 已确认差异

| 优先级 | 项目 | 当前实现 | 契约要求 | 说明 |
|---|---|---|---|---|
| 高 | LLM 默认网关 | `backend/app/config.py` 的默认 `LLM_ENDPOINT` 仍是 `https://genaiapi.shanghaitech.edu.cn/api/v1/start`，默认模型为 `deepseek-v3:671b` | 所有大模型调用必须经由 `http://hospital-llm-gateway:8080/v1/chat/completions`，且模型必须落在白名单内 | 运行是否合规依赖部署时覆盖环境变量，仓库默认值本身不合规 |
| 高 | 模型白名单校验 | patient / triage / doctor 相关 agent 只把 `llm_settings.model` 透传给请求层，没有做白名单强校验 | 请求 body 中的 `model` 必须为契约白名单项 | 需要在启动或请求前做硬校验，避免误配模型 |
| 中 | 非 `/api/v1` 调试入口 | `GET /patient-agent-debug` 仍然存在 | 所有对外 HTTP 接口必须使用 `/api/v1/` 前缀 | 该页面是 HTML 调试页，不是业务 JSON 接口，但仍属于对外 HTTP 暴露面 |
| 中 | 契约自检缺口 | 尚未看到专门覆盖 patient agent / triage / doctor 的契约 smoke test | 应当在 CI 中验证 API 前缀、LLM 网关、模型白名单、响应壳 | 目前更多依赖人工检查，后续容易回退 |

## 关联说明

- patient agent 的 JSON 调试接口已经使用 `/api/v1/patient-agent-debug/*`，并且返回 `ok/data` 外壳，这部分是合规的。
- 当前的 LLM 配置入口被分诊、内科和 ICU 共享，因此 `config.py` 的默认值差异会同时影响这三条链路。
- patient agent 本体已经是结构化、受控的病例卡 / 回复生成器，不是开放式聊天空壳；这里记录的是契约层面的差异，不是否认功能实现。

## 建议整改顺序

1. 把默认 LLM endpoint 改为教学组要求的网关地址，并把默认模型换成白名单中的值。
2. 在启动阶段或请求前加入模型白名单校验，避免误配。
3. 处理或降级 `/patient-agent-debug` 这类非 `/api/v1` HTML 暴露面。
4. 增加一组最小契约测试，专门覆盖 LLM 网关、接口前缀和响应壳。

## 备注

如果后续要把这份存档进一步转成整改计划，建议按“硬性契约项优先、调试入口次之、测试补齐最后”的顺序推进。
