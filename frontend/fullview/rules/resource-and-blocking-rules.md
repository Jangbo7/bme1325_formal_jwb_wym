# Resource and Blocking Rules

本文件保存全院通用资源、队列、阻塞和异常规则。所有部门事件都应遵守这些规则。

## ICU 满床阻塞

- 事件 ID: `BLOCK_ICU_FULL`
- 所属分类: 资源阻塞
- 触发条件: 任意科室请求转入 ICU，但 `icu_beds_a`、`icu_beds_b`、`icu_isolation`、`intervention_bay` 没有匹配空床。
- 涉及房间: ICU 床位房间、请求来源房间。
- 涉及人员/Agent: ICU 医生、BedManager、发起医生、患者。
- 前置检查: 已查询 ICU 床位库存和床位类型。
- 执行动作: 拒绝或挂起转入请求，记录等待原因。
- 成功后的状态/资源变化: 患者留在原房间；原床位不释放；ICU 床位不变化。
- 阻塞或失败提示: “ICU 无可用床位，转运阻塞，患者留原科室。”
- 可视化表现: ICU snapshot 显示 critical，患者不移动。

## 住院无床阻塞

- 事件 ID: `BLOCK_WARD_FULL`
- 所属分类: 资源阻塞
- 触发条件: 门诊、急诊或 ICU 请求转入住院，但目标专科无匹配床位。
- 涉及房间: `ward_admission`、各 `*_ward`、请求来源房间。
- 涉及人员/Agent: BedManager、AdmissionNurse、发起医生、患者。
- 前置检查: 已按专科、年龄、性别、隔离需求查询床位。
- 执行动作: 患者进入待入院或待转出队列。
- 成功后的状态/资源变化: 患者留原科室；原床位不释放；住院床位不变化。
- 阻塞或失败提示: “住院部无匹配床位，患者进入待入院队列。”
- 可视化表现: 住院部 snapshot 显示 warning，患者留原房间。

## 急诊抢救床满

- 事件 ID: `BLOCK_ED_RESUS_FULL`
- 所属分类: 资源阻塞
- 触发条件: L1/L2 患者需要进入 `ed_red_resus`，但抢救床已满。
- 涉及房间: `ed_triage`、`ed_red_resus`、`ed_waiting`
- 涉及人员/Agent: 分诊护士、急诊医生、床旁护士、患者。
- 前置检查: 已查询抢救床占用。
- 执行动作: 患者留在分诊或临时抢救区域，触发急诊告警。
- 成功后的状态/资源变化: 患者状态可为 `IN_TREATMENT`，但房间不转入 `ed_red_resus`。
- 阻塞或失败提示: “抢救床满，需要急诊医生人工协调。”
- 可视化表现: `ed_red_resus` 显示 critical。

## 诊室占用阻塞

- 事件 ID: `BLOCK_CONSULT_ROOM_BUSY`
- 所属分类: 队列阻塞
- 触发条件: 候诊患者准备进入诊室，但目标诊室仍被占用。
- 涉及房间: `outpatient_waiting`、门诊诊室、`ed_waiting`、急诊诊疗房间。
- 涉及人员/Agent: 患者、医生、叫号系统。
- 前置检查: 已检查目标诊室占用状态。
- 执行动作: 患者继续留在 waiting room，等待下一次叫号。
- 成功后的状态/资源变化: waiting room 人数不变，诊室状态不变。
- 阻塞或失败提示: “目标诊室正在使用，患者继续候诊。”
- 可视化表现: 患者不移动，诊室保持 occupied。

## 检查排队

- 事件 ID: `BLOCK_EXAM_QUEUE_BUSY`
- 所属分类: 检查队列
- 触发条件: 患者需要检查，但 `lab_2`、`ed_diagnostic`、`diagnostic_center` 或 `imaging_review` 正在服务其他任务。
- 涉及房间: `lab_2`、`ed_diagnostic`、`diagnostic_center`、`imaging_review`
- 涉及人员/Agent: 患者、检查人员、医生、护士。
- 前置检查: 检查医嘱有效，患者身份核验通过。
- 执行动作: 将患者加入检查等待队列。
- 成功后的状态/资源变化: 患者可保持原房间或移动到等待区；检查房不重复占用。
- 阻塞或失败提示: “检查资源占用，患者进入检查等待队列。”
- 可视化表现: 检查房显示 busy，患者等待。

## 药房窗口或库存阻塞

- 事件 ID: `BLOCK_PHARMACY_UNAVAILABLE`
- 所属分类: 药房阻塞
- 触发条件: 患者需要取药或 ICU 需要用药，但窗口占用、处方缺失、未支付或库存不足。
- 涉及房间: `pharmacy_2`、`icu_medication`
- 涉及人员/Agent: 患者、药师、护士、医生。
- 前置检查: 处方、支付、库存、药物审核。
- 执行动作: 阻塞发药或给药，提示具体原因。
- 成功后的状态/资源变化: 药品库存不变；患者不进入完成状态。
- 阻塞或失败提示: “药房暂不可服务或药品不可用。”
- 可视化表现: 药房显示 warning。

## 身份核验失败

- 事件 ID: `BLOCK_IDENTITY_VERIFICATION_FAILED`
- 所属分类: 患者安全
- 触发条件: 给药、检查、转运、出院或入科接收前身份核验失败。
- 涉及房间: 任意患者当前房间。
- 涉及人员/Agent: 患者、护士、医生、AdmissionNurse。
- 前置检查: 需要至少两个身份字段核验。
- 执行动作: 禁止执行目标动作，记录安全事件。
- 成功后的状态/资源变化: 患者状态和资源占用不变。
- 阻塞或失败提示: “身份核验失败，禁止执行该动作，需要人工确认。”
- 可视化表现: 患者当前位置显示 safety warning。

## 接收科室拒收

- 事件 ID: `BLOCK_RECEIVER_REJECTED`
- 所属分类: 跨部门阻塞
- 触发条件: 目标科室收到转运或会诊请求后拒绝接收。
- 涉及房间: 请求来源房间、目标部门入口房间。
- 涉及人员/Agent: 发起医生、接收科室医生、BedManager、患者。
- 前置检查: 已发送 handoff 或转运请求。
- 执行动作: 转运取消或挂起，记录拒收原因。
- 成功后的状态/资源变化: 患者留原科室；原资源不释放；目标资源不占用。
- 阻塞或失败提示: “目标科室拒收，请补充资料或重新评估。”
- 可视化表现: 患者不移动，目标部门显示 rejected/warning。

## 转运人员不足

- 事件 ID: `BLOCK_TRANSPORT_STAFF_UNAVAILABLE`
- 所属分类: 转运阻塞
- 触发条件: 跨楼层或危重患者转运需要陪护医生、护士、担架或设备，但资源不足。
- 涉及房间: 患者当前房间、目标房间、对应电梯房间。
- 涉及人员/Agent: 陪护医生、护士、患者、转运人员。
- 前置检查: 已锁定目标床位或目标房间；已查询转运资源。
- 执行动作: 保留目标资源一段时间或取消锁定，患者等待转运人员。
- 成功后的状态/资源变化: 患者状态可为 `TRANSFERRING` 的等待子状态；房间不移动。
- 阻塞或失败提示: “转运人员不足，等待陪护或担架资源。”
- 可视化表现: 患者留原房间，转运路径不启动。

## 患者状态不稳定阻塞普通转运

- 事件 ID: `BLOCK_UNSTABLE_FOR_ROUTINE_TRANSFER`
- 所属分类: 患者安全
- 触发条件: 患者生命体征不稳定，但事件尝试进行普通转运。
- 涉及房间: 患者当前房间、目标房间。
- 涉及人员/Agent: 医生、护士、Risk Sentinel 或分诊 Agent、患者。
- 前置检查: 最近生命体征、风险评分、医生评估。
- 执行动作: 阻止普通转运，改走急诊或 ICU 绿色通道。
- 成功后的状态/资源变化: 患者留原位置；可能触发 `alert.raised`。
- 阻塞或失败提示: “患者状态不稳定，禁止普通转运。”
- 可视化表现: 患者不移动，当前位置显示 critical。

## 资料缺失阻塞 MDT 或转诊

- 事件 ID: `BLOCK_MISSING_CLINICAL_DATA`
- 所属分类: 资料阻塞
- 触发条件: MDT、转 ICU、转住院或出院需要关键资料，但病历、检查或影像缺失。
- 涉及房间: `mdt_call`、`imaging_review`、`ed_handoff`、`icu_transfer`、`ward_admission`
- 涉及人员/Agent: 发起医生、接收医生、MDT_Call Agent、患者。
- 前置检查: 已检查病例摘要、检查结果、影像资料、待办任务。
- 执行动作: 挂起请求，生成补充资料任务。
- 成功后的状态/资源变化: 患者保留原位置；目标队列可保留待补资料状态。
- 阻塞或失败提示: “关键临床资料缺失，无法完成转诊或会诊。”
- 可视化表现: 对应入口房间显示 warning。

## 危急值未签收

- 事件 ID: `BLOCK_CRITICAL_RESULT_UNSIGNED`
- 所属分类: 患者安全
- 触发条件: 检查中心产生危急值，但医生或护士未在规定时间内签收。
- 涉及房间: `diagnostic_center`、`ed_diagnostic`、`lab_2`、患者所在房间。
- 涉及人员/Agent: 检查中心、医生、护士、患者。
- 前置检查: 检查结果已标记 critical；签收超时。
- 执行动作: 触发升级提醒，通知医生和护士长。
- 成功后的状态/资源变化: 患者状态可保持不变，但部门 alert level 升级。
- 阻塞或失败提示: “危急值未签收，需要立即确认。”
- 可视化表现: 检查房和患者房间高亮 critical。

## 电梯路径不可用

- 事件 ID: `BLOCK_ELEVATOR_PATH_UNAVAILABLE`
- 所属分类: 路径阻塞
- 触发条件: 跨楼层转运需要电梯，但起点或终点电梯路径不可用。
- 涉及房间: `elevator_1`、`elevator_2`、`elevator_3`、`elevator_4`、`elevator_5`
- 涉及人员/Agent: 患者、转运人员、可视化运行时。
- 前置检查: 已计算起点到电梯和电梯到目标房间路径。
- 执行动作: 暂停转运，提示路径配置或碰撞配置问题。
- 成功后的状态/资源变化: 患者留原房间；目标资源可释放或保持锁定。
- 阻塞或失败提示: “跨楼层路径不可用，无法启动转运。”
- 可视化表现: 不启动移动动画，room readout 显示路径失败。

## 床位一致性错误

- 事件 ID: `ERROR_BED_STATE_INCONSISTENT`
- 所属分类: 资源错误
- 触发条件: 患者记录、床位库存和可视化房间占用不一致。
- 涉及房间: ICU 或住院任意床位房间。
- 涉及人员/Agent: BedManager、护士长、系统管理员。
- 前置检查: 已进行床位对账。
- 执行动作: 阻止新的床位分配或释放，要求人工修正。
- 成功后的状态/资源变化: 系统进入 `ERROR` 或部门 warning 状态。
- 阻塞或失败提示: “床位状态不一致，需要人工校正。”
- 可视化表现: 对应床位或部门 snapshot 显示 error。
